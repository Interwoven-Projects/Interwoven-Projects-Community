# Contributing to Interwoven Projects

Thank you for your interest in collaborating! This document explains the practical ways to participate.

## The Spirit of This Community

- We are interdisciplinary by design. Respect different ways of knowing and working.
- Credit matters. We make contributions visible and portable.
- We default to kindness, clarity, and generosity.
- Anyone can propose; anyone can contribute according to their strengths.

## Quick Ways to Contribute

| What you want to do                  | How to do it                                                                 |
|--------------------------------------|-------------------------------------------------------------------------------|
| Introduce yourself & your skills     | [Open a Discussion](https://github.com/Interwoven-Projects-Community/Interwoven-Projects-Community/discussions) in "Introductions & Skills" |
| Propose a new project                | Use the [Project Proposal form](https://github.com/Interwoven-Projects-Community/Interwoven-Projects-Community/issues/new?template=project-proposal.yml) |
| Join an existing project             | Comment on its thread or use the [Join Project form](https://github.com/Interwoven-Projects-Community/Interwoven-Projects-Community/issues/new?template=join-project.yml) |
| Record a contribution you made       | Use the [Contribution Update form](https://github.com/Interwoven-Projects-Community/Interwoven-Projects-Community/issues/new?template=contribution-update.yml) |
| Offer feedback or help               | Comment thoughtfully on active Discussions and Issues                         |
| Improve these guidelines             | Open a PR or Discussion suggesting changes                                    |

## Project Lifecycle (Simplified)

1. **Proposal** → Structured issue using the template
2. **Discussion** → Happens in GitHub Discussions + the proposal issue
3. **Activation** → When a small team forms and commits to moving forward, the project gets an entry under `/projects`
4. **Execution** → Tracked via issues, updates, or linked repositories
5. **Credit & Completion** → CONTRIBUTORS.md is finalized; work is shared

## Labels We Use

We use labels heavily so people can find relevant work quickly.

**Field labels** (examples):
- `field:ai-ml`
- `field:physics`
- `field:history`
- `field:philosophy`
- `field:coding-software`
- `field:biology`
- `field:writing`
- `field:design`

**Status labels**:
- `status:seeking-contributors`
- `status:needs-lead`
- `status:active`
- `status:paused`
- `status:completed`

**Needs labels**:
- `needs:developer`
- `needs:writer`
- `needs:domain-expert`
- `needs:researcher`
- `needs:designer`
- `needs:reviewer`

**Other useful labels**:
- `beginner-friendly`
- `credit-open`
- `good-first-issue` (for repo maintenance)

You can filter issues and discussions using these labels.

## Issue Templates

We have three main forms:

- **Project Proposal** — For new ideas (required fields for clarity)
- **Join Project** — Express interest in a specific project
- **Contribution Update** — Record work you've done so it can be credited

Using the templates helps everyone. Please try to fill them out thoughtfully.

## Pull Requests

Pull requests are welcome for:
- Improving documentation in this hub repo
- Code or content for active projects (when the project uses this repo or a linked one)
- Fixes to templates, labels, or guidelines

For project-specific work, follow the conventions of that project (some may live in separate repositories).

## Communication Norms

- Be patient — many contributors have day jobs and other projects.
- When giving feedback, be specific and kind.
- If you're unsure about credit or scope, ask early in the thread.
- Tag people thoughtfully (avoid @-everyone unless truly necessary).

## Code of Conduct

All participants are expected to follow our [Code of Conduct](./CODE_OF_CONDUCT.md). Violations can be reported to the maintainers via private message or the mechanisms described in the CoC.

## Questions?

- General questions → GitHub Discussions
