# Attribution Guidelines (Companion)

This is a short companion to the main [ATTRIBUTION.md](../ATTRIBUTION.md).

## Quick Rules of Thumb

- If someone did real work that moved the project forward, they get credited.
- Use the standard roles table when possible.
- Record contributions as they happen using the **Contribution Update** issue form.
- The Project Lead is ultimately responsible for maintaining the CONTRIBUTORS section, but everyone helps by submitting updates.

## Recommended Credit Placement

| Project Stage     | Where to put credit                          |
|-------------------|----------------------------------------------|
| Early discussion  | In the main proposal issue / discussion      |
| Active project    | `CONTRIBUTORS.md` or "Contributors" section in README |
| Completed work    | Final artifact + permanent link to the credit record |

## Updating Credit

Use the [Contribution Update form](../.github/ISSUE_TEMPLATE/contribution-update.yml). This creates a clean paper trail.

## Common Credit Mistakes to Avoid

- Forgetting people who did "small" but important work (reviewing, connecting people, asking the right question at the right time).
- Over-crediting the originator at the expense of those who did the actual heavy lifting.
- Letting the CONTRIBUTORS list get stale.

## Philosophy

We treat credit the way good scientists treat authorship: it should be accurate, generous where appropriate, and defensible.
