# Collaboration Tools & Resources

A living collection of tools and practices that help interdisciplinary teams work well together.

## Communication & Discussion

- **GitHub Discussions & Issues** — Primary home for this community (asynchronous, threaded, searchable)
- **X (Twitter)** — Fast discovery and casual conversation. Use the community account and consistent hashtags when appropriate.
- **Discord / Slack / Matrix** (optional per project) — Good for real-time chat once a small team forms. Create a dedicated space only when needed. **All meetings for projects need to be approved by the Moderator/Project Lead of the specific project being worked on.** Discord: https://discord.gg/4QRR2y3QR

## Shared Knowledge & Documents

- **Notion, Obsidian Publish, or GitHub Wiki** — For living project notes
- **Google Docs / HackMD / HedgeDoc** — Quick collaborative writing
- **Overleaf** — For serious academic writing with LaTeX
- **Figma** — Diagrams, visual explanations, and collaborative design (free tier is often enough)

## Version Control & Code

- This GitHub repository (hub)
- Dedicated project repositories when the work outgrows a single folder
- Good commit messages and clear READMEs are non-negotiable for cross-disciplinary legibility

## Data, Models & Experiments

- **OSF (Open Science Framework)** — Excellent for research projects that need preregistration, data, and materials
- **Zenodo** — For DOIs and long-term archiving of datasets, papers, and software
- **Jupyter notebooks + GitHub** — Great for reproducible analysis that spans fields

## Visual & Narrative Tools

- **Excalidraw, Miro, or Whimsical** — For quick collaborative diagrams
- **Canva** — Accessible visual design for non-designers
- **Tldraw** — Lightweight infinite canvas

## Project Management (lightweight)

- GitHub Projects board (already set up for the community)
- Simple shared issue labels within a project repo
- A single "Status" thread or document that is updated weekly

## Attribution & Credit Tools

- The forms in this repository (Contribution Update issues)
- `CONTRIBUTORS.md` files
- CRediT taxonomy (https://credit.niso.org/) — useful when projects produce formal academic outputs

## Recommended Practices for Interdisciplinary Work

1. **Over-communicate assumptions** — What seems obvious in one field can be completely foreign in another.
2. **Create glossaries** — A living "Terms" document saves enormous amounts of time.
3. **Write for the interested outsider** — The best outputs explain enough for a smart person from another discipline to understand the core ideas.
4. **Record decisions** — Use dated notes or decision logs.
5. **Celebrate small wins publicly** — It keeps energy high across long projects.

## Want to suggest a tool?

Open a Discussion or PR. We're especially interested in tools that are accessible to people without institutional logins or big budgets.
